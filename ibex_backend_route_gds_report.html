Hypothesis

DRC matrix inspection에서 모든 violation이 M1/M2/VIA1에 집중되어 있었다. 특히 M1-M2 Needs fat contact가 347개로 가장 컸다. 이 관찰은 root cause를 global routing, timing, PG보다 lower-metal physical abstract, contact/via definition, standard-cell pin-access policy 쪽으로 이동시켰다.

진단 원칙. DRC total만 보지 않고 class와 layer matrix를 봐야 한다. total 720은 너무 크지만, M1/M2/VIA1에 집중된다는 사실이 해결 방향을 만든다.

NDM build command 흐름

NDM은 ICC2가 읽는 physical library format이다. 이 프로젝트는 먼저 techfile patch를 만들고, 그 techfile과 modified LEF, timing DB를 Library Manager가 workspace로 읽은 뒤 NDM으로 commit했다.

Shell wrapper:
4_Backend_ICC2/0_Script/99_debug/build_via1_pitch_no_track_ndm.sh

Flow:
1. base techfile 복사 및 VIA1 patch 생성
2. MOD_LEF_TECH_FILE, MOD_LEF_NDM_DIR, MOD_LEF_NDM_SUFFIX environment 설정
3. lm_shell -f 4_Backend_ICC2/0_Script/99_debug/build_modified_lef_ndm.tcl
4. RVT/LVT/HVT 각각 create_workspace -> read_db -> read_lef -> check_workspace -> commit_workspace

Core LM commands:

read_db $RVT_TT_DB
read_lef $RVT_LEF
check_workspace
commit_workspace -output $NDM_DIR/saed32rvt_tt.modified_lef_via1_pitch_no_track.ndm -force
왜 승인이 필요했는가. LEF/techfile을 바꿔 route DRC를 없애는 것은 강한 조치다. 학습용 project에서는 허용할 수 있지만, 실무에서는 PDK/library owner 승인, regression, signoff tool consistency 확인 없이 조용히 섞으면 안 된다.
Stage gate. modified LEF와 VIA1 no-track NDM build가 fatal 없이 완료되고, route/FM evidence와 policy 문서가 남았기 때문에 debug-only result를 named baseline으로 승격할 수 있었다.

11. Practical Judgement And Claim Boundary

이 프로젝트의 backend 결과는 학습용으로는 강한 완료 상태다. route DRC, electrical DRC, legality, PG, FM, PT, GDS export가 모두 저장 artifact로 연결되어 있다. 다만 이것은 tapeout signoff package가 아니다.

말할 수 있는 것 아직 말하면 안 되는 것
ICC2 educational GDS candidate가 생성되었다. tapeout-ready GDS다.
after-filler ICC2 route DRC/open nets는 clean report다. foundry signoff DRC clean이다.
PG connectivity/PG DRC reports가 clean하다. IR/EM signoff가 완료됐다.
post-route source netlist가 Formality를 통과했다. LVS clean이다.
PrimeTime SDF STA에서 setup/hold violation이 없다. multi-corner multi-mode signoff STA가 완료됐다.
antenna rules가 없어 antenna checking은 활성화되지 않았다. antenna clean이다.

Interview explanation

첫 route는 연결성과 timing은 양호했지만 720개 signal DRC가 있었다. DRC matrix를 분석해 M1/M2/VIA1 lower-metal pin-access/contact 문제로 좁혔고, 단순 route iteration, M2-only reroute, route option, post-route via ECO, lower utilization, post-route resize를 하나씩 반증했다. Modified LEF가 fat-contact class를 제거했고, VIA1 pitch/no-track techfile과 upstream cell-use policy가 Off-grid/MUX41 pin-access 문제를 제거했다. 이후 policy approval을 받아 named baseline으로 승격했고, GDS export 후 발견된 max transition/cap issue는 route_opt, ECO, cleanup, pre-filler margin ECO 순서로 닫았다. 최종 후보는 교육용으로는 clean하지만, signoff DRC/LVS/antenna/IR/EM/metal fill이 없으므로 production/tapeout claim은 하지 않는다.
