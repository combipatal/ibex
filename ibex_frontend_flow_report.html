1. Project Overview

이 프로젝트의 목적은 Ibex RISC-V core를 단순 RTL 데모로 끝내지 않고, 실제 ASIC 구현 flow의 기본 증거 체인까지 연결하는 것이다. 첫 번째 milestone은 RTL intake, source revision freeze, configuration freeze, Mini SoC top 구성, DC synthesis, Formality R2N, PT STA, ICC2 backend, post-route report collection, GDS candidate export를 하나의 재현 가능한 flow로 만드는 것이다.

front-end에서 중요한 점은 합성이 끝났다는 사실보다 “어떤 RTL, 어떤 parameter, 어떤 top, 어떤 filelist, 어떤 constraint로 합성했는지”가 backend와 Formality에 동일하게 전달되는 것이다. Ibex는 parameter에 따라 hierarchy, port, debug/RVFI/counter/cache logic, timing, backend size가 달라지므로 configuration freeze 없이 결과를 비교하면 debug가 불가능해진다.

3. Mini SoC Top Construction

rtl/mini_soc는 Ibex core를 synthesis/backend 대상 top으로 만들기 위한 최소 SoC wrapper이다. 목표는 완전한 product SoC가 아니라 front-end부터 backend까지 관찰 가능한 표준-cell-only design을 만드는 것이다.

Mini SoC block diagram

ibex_mini_soc_top
  |
  +-- u_ibex_top
  |     +-- instruction bus: instr_req/gnt/rvalid/addr/rdata
  |     +-- data bus       : data_req/gnt/rvalid/we/be/addr/wdata/rdata
  |     +-- timer_irq input, debug/RVFI/cache/security tie-offs
  |
  +-- u_imem        instruction memory, preload interface, boot at 0x0000_0000
  +-- u_decoder     data address decode
  +-- u_dmem        data memory target
  +-- u_gpio        GPIO peripheral target
  +-- u_timer       timer peripheral and irq_o
  +-- unmapped path returns data_err
Module 역할 학습 포인트
ibex_mini_soc_top.sv clock/reset, Ibex instance, bus mux, top IO 실제 synthesis top이며 parameter/tie-off가 모이는 위치
simple_imem.sv instruction memory 및 preload path boot address 0x0에서 instruction fetch가 가능하도록 구성
simple_dmem.sv data memory load/store bus의 기본 read/write handshake 제공
simple_decoder.sv address decode DMEM/GPIO/timer/unmapped access 분기
gpio.sv, timer.sv basic peripheral software-visible peripheral과 timer interrupt path 제공

Module 구성 강화 표

Block 주요 interface 합성/backend 관점에서 중요한 이유
u_ibex_top clk_i, rst_ni, instruction/data bus, interrupt/debug tie-offs core configuration이 실제 gate count, hierarchy, timing, Formality compare point를 결정한다.
u_imem Ibex instruction bus + external preload pins memory macro 없이 stdcell memory로 first backend baseline을 만든다.
u_decoder data_addr 입력, select outputs data bus fanout과 peripheral select path를 만들어 synthesis/timing 대상이 된다.
u_dmem data req/we/be/addr/wdata/rvalid/rdata load/store path의 main sequential logic이며 backend cell count에 크게 반영된다.
u_gpio memory-mapped data bus, gpio_i/gpio_o top-level IO timing constraint와 연결되는 simple peripheral이다.
u_timer memory-mapped data bus, timer_irq Ibex timer interrupt path를 실제 연결해 interrupt tie-off policy를 검증 가능하게 한다.
unmapped response unmapped_rvalid_q, data_err decode miss가 X로 떠다니지 않고 deterministic error response가 되도록 한다.

Ibex instruction bus는 instr_req/gnt/rvalid/addr/rdata 형태로 IMEM에 연결되고, data bus는 decoder를 거쳐 DMEM, GPIO, timer, unmapped response로 나뉜다. unmapped access는 data_err를 통해 error response를 만들도록 구성했다.

왜 custom wrapper가 중요한가. debug, RVFI, scramble, integrity, unused alert/shadow output을 synthesis script에서 상수로 묶으면 DC와 FM의 elaboration 차이가 생기기 쉽다. wrapper RTL에서 tie-off하면 filelist만 맞춰도 동일한 구조를 재현할 수 있다.
Stage gate. Mini SoC top이 Ibex instruction/data interface, peripheral response, interrupt/tie-off를 모두 닫았기 때문에 DC analyze/elaborate에서 dangling top-level core interface 없이 합성 가능한 단일 top으로 넘어갈 수 있었다.

R2N에서 배운 점

  • SVF는 단순한 보조 파일이 아니라, DC optimization과 hierarchy mapping을 Formality가 이해하는 데 필요한 provenance이다.
  • Ibex parameter를 DC script에서만 override하면 FM reference와 implementation이 다른 design을 볼 수 있다. 그래서 parameter는 wrapper RTL에 둔다.
  • Formality가 느릴 때는 임의로 종료 후 rerun하는 것보다 accepted/rejected SVF guidance, unmatched compare point, read/link warning부터 확인해야 한다.
Stage gate. reference RTL과 mapped implementation이 34915 passing compare point, 0 failing, 0 unmatched로 검증되었기 때문에 netlist를 backend와 STA 대상으로 사용할 논리적 근거가 생겼다.

7. PrimeTime Pre-Backend STA

PrimeTime pre-backend STA는 DC topo netlist, generated SDC, generated SDF를 읽어 backend 이전 handoff timing을 확인했다. 여기서 SVF는 입력이 아니다. SVF는 Formality provenance이고, STA는 netlist/SDC/SDF/library를 기준으로 한다.

왜 같은 filelist/config를 고집했는가

Ibex의 ibex_top parameter가 달라지면 generated hierarchy와 port set, counter/debug/cache logic, backend instance count가 달라진다. DC와 FM이 다른 해석을 하면 R2N은 tool 문제가 아니라 입력 불일치로 깨진다. 이 프로젝트는 wrapper RTL에서 parameter를 고정하고, DC/FM reference filelist alignment를 문서화해 그 위험을 줄였다.

Stage gate. netlist/SDC/SDF/SVF와 pass/fail reports가 한 run tag로 묶였기 때문에 backend team 또는 context reset 이후 agent가 동일 handoff를 재사용할 수 있다.

9. Lessons Learned

  1. config freeze가 먼저다. “simple config”라는 말만으로는 결과를 재현할 수 없다. RV32E/I, RV32M, PMP, debug, RVFI, counters, ICache, branch option, reset vector, interrupt tie-off까지 값으로 기록해야 한다.
  2. top과 integration point를 명시해야 한다. ibex_core와 ibex_top은 port와 wrapper responsibility가 다르다. 실제 synthesis top과 Ibex instance path를 문서화해야 backend debug가 쉬워진다.
  3. DC completion은 success가 아니다. QoR, constraints, check_design, generated outputs, SVF, report warning을 같이 봐야 한다.
  4. SVF는 Formality의 핵심 입력이다. hierarchy mapping guidance가 없으면 FM이 느리거나 불안정해질 수 있다.
  5. PT에서 SVF를 읽지 않는다. STA는 netlist/SDC/SDF/library 기반이고, SVF는 logic equivalence provenance이다.
  6. reset policy는 claim boundary를 만든다. rst_ni false path는 baseline을 단순화하지만 recovery/removal 검증을 수행한 것은 아니다.
